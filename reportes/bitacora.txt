  DESCRIPCIÓN GENERAL DEL PROYECTO
================================================================================

Proyecto académico de Programación Numérica orientado al análisis de datos
climáticos reales registrados por dos estaciones meteorológicas en El Salvador:

  · EEP  — Estación exterior (escuela / zona 1)
  · UES  — Estación exterior (universidad / zona 2)

Restricciones académicas obligatorias:
  - Todos los cálculos estadísticos deben implementarse de forma nativa
    (sin .mean(), .std(), .median(), .min(), .max(), .quantile() de pandas/numpy)
  - El motor matemático se implementa en C++ y se expone a Python mediante ctypes
  - Las gráficas interactivas deben renderizarse con Canvas (HTML5 / uPlot / Chart.js)
    sin depender de matplotlib para el output final al usuario

  · _percentil(s, p)   — percentil por interpolación lineal post-sort
  · _pearson(x, y)     — correlación de Pearson sin caja negra
  · _estadisticos()    — función que reúne todos los anteriores (n, media,
                         varianza, σ, máx, mín, rango, p10, p25, p50, p75, p90,

================================================================================
  NOTAS TÉCNICAS Y DECISIONES DE DISEÑO
================================================================================

1. Comparativa de rangos (ambos módulos)
   - Se eligió mostrar la comparativa con rango de FECHAS (date picker) en lugar
     de selector por día/mes/año porque permite comparar períodos arbitrarios
   - Los rangos A y B se codifican en color: rojo (#f87171) vs verde (#34d399)
   - Las mini-gráficas Canvas downsamplea la serie a 60 puntos para rendimiento
   - La interpretación automática analiza: diferencia de medias, diferencia de σ,
     y advierte si algún rango tiene pocos datos (N < 10)
   - Modo comparación y modo calendario son mutuamente excluyentes (toggle)

2. Por qué uPlot en vez de Chart.js para las series temporales
   - uPlot es ~100x más rápido que Chart.js para series de miles de puntos
   - Usa WebGL-level Canvas rendering con repaint mínimo
   - Soporta sincronización de cursor entre múltiples gráficas (key: 'clima-sync')
   - Chart.js se mantiene para histogramas y boxplots donde uPlot no encaja

3. Caché analisis_cache.pkl
   - Evita recalcular 154,655 registros de SMA + los CSVs climáticos en cada run
   - Se invalida si el mtime de algún CSV cambia
   - Es de 103 MB porque incluye el DataFrame completo serializado

4. Interpolación de datos faltantes
   - Se usa interpolación lineal manual (sin .interpolate() de pandas)
   - Solo se interpola si el gap es corto (configurable, default ≤ 3 muestras)
   - Gaps más largos se dejan como NaN para no inventar datos

5. Datos del SMA
   - Los CSV del inversor SMA tienen 58 columnas y separador configurable (; o ,)
   - La columna pac_total se calcula sumando pac1 + pac2 + pac3
   - La energía por intervalo = pac_total × Δt_min / 60 (en Wh)
   - Los archivos duplicados (nombre(1).csv) se fusionan con el principal
